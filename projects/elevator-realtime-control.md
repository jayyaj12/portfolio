## Overview
Android 기반 엘리베이터 디스플레이·제어 시스템을 단독 개발·운영한 프로젝트로,  
RS-232 기반 시리얼 통신과 UDP 네트워크 통신을 통해  
**실시간 상태 수신·표시·제어**를 안정적으로 수행하는 것을 목표로 했습니다.  

현장 환경 특성상 네트워크 품질이 불안정하고 장애 대응이 즉각적으로 요구되었기 때문에,  
본 프로젝트에서는 **통신 안정성, 자동 복구, 무중단 운영**에 초점을 맞춰 설계·개선했습니다.

---

## Context
- 기간: 2023.06 ~ 2025.06  
- 역할: Android Developer (단독 개발 및 운영)  
- 기술 스택: Kotlin, Android, Serial(RS-232), UDP, Coroutine, OTA Update, Remote Reboot  

---

## Problem
- 엘리베이터 제어 특성상 **실시간성 지연이 사용자 경험에 직접적인 영향**
- RS-232 통신 특성으로 인한 데이터 손실 및 노이즈 이슈
- 네트워크 장애 발생 시 현장 출동에 의존하는 구조
- 디바이스 장애 발생 시 원격 조치 수단 부족
- 운영 중 로그 부족으로 장애 원인 분석이 어려움

---

## Approach

### 1. 통신 구조 설계
- RS-232 기반 시리얼 통신을 통해 엘리베이터 제어 신호 수신
- UDP 통신을 활용해 중앙 서버와 상태 정보 송수신
- 패킷 단위 데이터 파싱 및 유효성 검증 로직 구현

### 2. 실시간 UI 반영 구조
- Coroutine 기반 비동기 처리로 통신 스레드와 UI 스레드 분리
- 수신된 상태 데이터를 즉시 UI에 반영해 지연 최소화
- 상태 변경 이벤트 중심으로 화면 갱신 구조 설계

### 3. 안정성 및 장애 대응
- 통신 실패 감지 시 자동 재시도 로직 구현
- 네트워크 복구 시 상태 동기화 처리
- 디바이스 이상 상태를 서버로 즉시 보고하는 구조 설계

### 4. 운영 자동화 기능
- OTA(Update Over The Air) 업데이트 기능 구현
- 원격 재부팅 기능 도입으로 현장 출동 최소화
- 로그 수집 및 원격 분석을 통한 장애 원인 파악 지원

---

## Before / After

### Before
- 통신 장애 발생 시 수동 재부팅 또는 현장 조치 필요
- 상태 지연 및 UI 반영 불안정
- 장애 발생 원인 파악에 많은 시간 소요

### After
- 통신 오류 시 자동 복구 및 재시도 가능
- 실시간 상태 표시 안정화
- 원격 업데이트·재부팅을 통한 운영 효율 개선

---

## Result
- 엘리베이터 상태 정보 **실시간 반영 안정화**
- 원격 재부팅·OTA 도입으로 **현장 출동 빈도 감소**
- 운영 중 장애 발생 시 **평균 복구 시간 단축**
- 장기간 상용 환경에서 안정적으로 운영 중

---

## What I learned
- 실시간 시스템에서는 코드 구조보다 **운영 안정성이 최우선 가치**임을 경험
- 통신 장애는 “완전한 제거”보다 **빠른 감지와 복구**가 중요함
- Android 앱도 현장 설비와 결합되면 **임베디드에 가까운 관점**이 필요함
